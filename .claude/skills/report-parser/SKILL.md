# Report Parser skill

- This skill provides comprehensive reference material for the Atlantis PBEM game engine's turn order system, report format, and the mapping between them.
- When writing orders or trying to understand the game state, always load `rules/README.md` wiki - it has all information about game rules and orders

## Sub-files

- @report-structure.md - Structure of the report files, both text and JSON, with detailed field descriptions and examples. Load first for quick lookup.
- @report-reference.md — Complete report format reference (JSON and text report structures). Load when you need detailed field descriptions or parsing logic.

## Scripts

### Python (primary)

- `.claude/skills/report-parser/scripts/report_parser.py` — Python script to parse text reports into structured JSON data.
- `.claude/skills/report-parser/scripts/compare_reports.py` — Python script to compare text-parsed reports against JSON reports for validation.
- `.claude/skills/report-parser/scripts/tests/test_report_parser.py` — pytest unit tests for the parser.

**Running the parser:**
```bash
uv run .claude/skills/report-parser/scripts/report_parser.py <report-file>
# Example:
uv run .claude/skills/report-parser/scripts/report_parser.py game-examples/15/report.14
```
Outputs structured JSON to stdout.

**Running the comparison:**
```bash
uv run .claude/skills/report-parser/scripts/compare_reports.py [count] [--seed=N]
# Examples:
uv run .claude/skills/report-parser/scripts/compare_reports.py          # 10 random reports
uv run .claude/skills/report-parser/scripts/compare_reports.py 50       # 50 random reports
uv run .claude/skills/report-parser/scripts/compare_reports.py 999 --seed=42  # all reports, deterministic
```
Compares text parser output against JSON report files in `game-examples/`. Reports match percentage per report and overall summary.

**Running tests:**
```bash
uv run pytest .claude/skills/report-parser/scripts/tests/test_report_parser.py -v
```

## Quick Overview

Each game turn follows this cycle:
1. Players submit text order files (starting with `#atlantis`, ending with `#end`)
2. The engine validates orders (producing `orders-checked` files)
3. Orders execute in a fixed sequence (see orders-reference.md for full execution order)
4. Reports are generated per faction (JSON + text format)
5. An order template is included in the report for the next turn

## Key concepts:
- The map is built from hexagonal regions connected to each other, each with a terrain type, coordinates, population, tax, and optional structures and units. 
- Map leftmost side is connected to the rightmost side, but top is not connected to the bottom.
- Units have a name, number and items inside them (including men from different races), and can perform orders.
