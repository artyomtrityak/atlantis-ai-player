## 1. JSON Report Structure

The JSON report is the canonical data source. The text report is generated from the JSON report by `TextReportGenerator::output()`.

### 1.1 Top-Level Keys

All top-level keys in a player faction JSON report:

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `name` | string | Yes | Faction name (without trailing number) |
| `number` | integer | Yes | Faction number |
| `engine` | object | Yes | Engine and ruleset version info |
| `administrative` | object | Yes | Account/admin settings |
| `date` | object | Yes | Current game date |
| `type` | object | Conditional | Faction type points (if FACLIM_FACTION_TYPES) |
| `status` | object | Conditional | Faction limits and usage |
| `statistics` | array | Conditional | Treasury/ranking data (if FACTION_STATISTICS enabled) |
| `errors` | array | Conditional | Errors from this turn (omitted if empty) |
| `battles` | array | Conditional | Battle reports (omitted if no battles) |
| `events` | array | Conditional | Events from this turn (omitted if empty) |
| `attitudes` | object | Yes | Declared attitudes toward other factions |
| `unclaimed_silver` | integer | Yes | Silver in faction treasury |
| `skill_reports` | array | Yes | Newly visible skill descriptions |
| `item_reports` | array | Yes | Newly visible item descriptions |
| `object_reports` | array | Yes | Newly visible object/structure descriptions |
| `regions` | array | Yes | All visible regions with units and structures |

### 1.2 engine

```json
{
  "version": "5.2.5 (beta)",
  "ruleset": "NewOrigins",
  "ruleset_version": "3.0.0 (beta)",
  "json_report_version": "1.0.1"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `version` | string | Atlantis engine version |
| `ruleset` | string | Ruleset name |
| `ruleset_version` | string | Ruleset version |
| `json_report_version` | string | JSON report format version |

### 1.3 administrative

```json
{
  "email": "NoAddress",
  "password": "111",
  "password_unset": false,
  "show_unit_attitudes": false,
  "times_sent": true,
  "inactivity_deletion_turns": 3,
  "quit": "eliminated"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | string | Yes | Faction email address |
| `password` | string | Conditional | Password (omitted if `password_unset` is true) |
| `password_unset` | boolean | Yes | Whether password is still "none" |
| `show_unit_attitudes` | boolean | Yes | Whether to show attitude symbols on foreign units |
| `times_sent` | boolean | Yes | Whether The Times newsletter is being sent |
| `inactivity_deletion_turns` | integer | Optional | Turns remaining before auto-deletion (only when close to limit) |
| `quit` | string | Optional | Quit reason. Values: `"quit and restart"`, `"game over"`, `"won game"`, `"eliminated"` |

### 1.4 date

```json
{
  "month": "April",
  "year": 2
}
```

| Field | Type | Description |
|-------|------|-------------|
| `month` | string | Month name (January, February, ..., December) |
| `year` | integer | Game year |

### 1.5 type (Faction Types)

Present only when `FACTION_LIMIT_TYPE == FACLIM_FACTION_TYPES`. Each key present only if the faction has points in that type.

```json
{
  "martial": 1,
  "magic": 4
}
```

Possible keys: `"martial"`, `"war"`, `"trade"`, `"magic"`. Values are integers representing allocated points.

### 1.6 status

```json
{
  "regions": { "current": 3, "allowed": 10 },
  "mages": { "current": 4, "allowed": 5 },
  "apprentices": { "current": 0, "allowed": 10, "name": "Apprentice" },
  "quartermasters": { "current": 0, "allowed": 2 },
  "tacticians": { "current": 0, "allowed": 2 },
  "activity": { "current": 3, "allowed": 10 },
  "tax_regions": { "current": 3, "allowed": 10 },
  "trade_regions": { "current": 0, "allowed": 5 }
}
```

All sub-objects follow the pattern `{ "current": int, "allowed": int }` except `apprentices` which adds `"name"` (string).

| Field | Condition | Description |
|-------|-----------|-------------|
| `mages` | Always (with faction types) | Mage unit count vs limit |
| `apprentices` | If `APPRENTICES_EXIST` | Apprentice count vs limit |
| `regions` | If `FACTION_ACTIVITY == MARTIAL_MERGED` | Merged martial region activity |
| `activity` | If `FACTION_ACTIVITY != DEFAULT` and not merged | Activity cost vs limit |
| `tax_regions` | If `FACTION_ACTIVITY == DEFAULT` | Tax region count vs limit |
| `trade_regions` | If `FACTION_ACTIVITY == DEFAULT` | Trade region count vs limit |
| `quartermasters` | If `TRANSPORT` enabled | Quartermaster count vs limit |
| `tacticians` | If `TACTICS_NEEDS_WAR` | Tactician count vs limit |

### 1.7 statistics (Treasury)

Array of objects, one per item type the faction possesses. Only present if `FACTION_STATISTICS` is enabled.

```json
[
  {
    "amount": 6,
    "name": "leader",
    "plural": "leaders",
    "tag": "LEAD",
    "rank": 27,
    "max": 21,
    "total": 385
  }
]
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `amount` | integer | Yes | Quantity owned by this faction |
| `name` | string | Yes | Singular item name |
| `plural` | string | Yes | Plural item name |
| `tag` | string | Yes | 4-letter item abbreviation |
| `rank` | integer | Yes | Faction's rank (1 = most of this item) |
| `max` | integer | Yes | Most of this item any single faction has |
| `total` | integer | Yes | Total of this item across all factions |
| `illusion` | boolean | Optional | Present and true if item is illusory |

### 1.8 errors

Array of error objects. Each error has a message and optionally a unit reference.

```json
[
  {
    "message": "GIVE: Nonexistant target (2996).",
    "unit": { "name": "Guards", "number": 2171 }
  },
  {
    "message": "STUDY: Not enough funds.",
    "unit": { "name": "Builders", "number": 2896 }
  }
]
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `message` | string | Yes | Error message text |
| `unit` | object | Optional | Unit that caused the error |
| `unit.name` | string | - | Unit name |
| `unit.number` | integer | - | Unit number |

### 1.9 battles

Array of battle report objects. Only present if battles occurred this turn.

```json
[
  {
    "type": "battle",
    "report": [
      "Unit A (123) attacks Unit B (456) in forest (10,20) in Province!",
      "",
      "Attackers:",
      "Unit A (123), Faction (1), 10 humans [HUMN], combat 2.",
      "",
      "Defenders:",
      "Unit B (456), Faction (2), 5 orcs [ORC].",
      "",
      "Round 1:",
      "...",
      ""
    ]
  }
]
```

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | Either `"battle"` or `"assassination"` |
| `report` | array of strings | Lines of the battle report text. For assassinations seen by the defender faction, only the assassination text line is included. |

Battle report lines are pre-formatted text strings (not structured data). The report array always ends with an empty string (blank line).

### 1.10 events

Array of event objects. Events record things that happened during the turn.

```json
[
  {
    "category": "reward",
    "message": "Times reward of 200 silver."
  },
  {
    "category": "tax",
    "message": "Collects $550 in taxes in tundra (46,84) in Lipodou.",
    "unit": { "name": "Guards", "number": 1264 },
    "region": {
      "coordinates": { "label": "surface", "x": 46, "y": 84, "z": 1 },
      "province": "Lipodou",
      "terrain": "tundra"
    }
  }
]
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `message` | string | Yes | Event message text |
| `category` | string | Yes | Event category (see [Section 4](#4-event-categories)) |
| `unit` | object | Optional | Unit involved in the event |
| `unit.name` | string | - | Unit name |
| `unit.number` | integer | - | Unit number |
| `region` | object | Optional | Region where event occurred (basic_region_data) |

See [Section 4](#4-event-categories) for the full list of event categories.

### 1.11 attitudes

```json
{
  "default": "unfriendly",
  "hostile": [],
  "unfriendly": [
    { "name": "Creatures", "number": 2 }
  ],
  "neutral": [],
  "friendly": [],
  "ally": []
}
```

| Field | Type | Description |
|-------|------|-------------|
| `default` | string | Default attitude: `"hostile"`, `"unfriendly"`, `"neutral"`, `"friendly"`, or `"ally"` |
| `hostile` | array | Factions declared hostile |
| `unfriendly` | array | Factions declared unfriendly |
| `neutral` | array | Factions declared neutral |
| `friendly` | array | Factions declared friendly |
| `ally` | array | Factions declared ally |

Each faction entry: `{ "name": string, "number": integer }`.

### 1.12 skill_reports

Array of skill report objects. These are descriptions of skills that became newly visible this turn (through study or encounter).

```json
[
  {
    "name": "sailing",
    "tag": "SAIL",
    "level": 2,
    "description": "No skill report."
  }
]
```

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Skill name |
| `tag` | string | Skill abbreviation |
| `level` | integer | Skill level for this report |
| `description` | string | Full skill description text |

### 1.13 item_reports

Array of item report objects. Descriptions of items newly encountered.

```json
[
  {
    "name": "sword",
    "tag": "SWOR",
    "description": "A sword is a ..."
  }
]
```

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Item display name |
| `tag` | string | Item abbreviation |
| `description` | string | Full item description text |

### 1.14 object_reports

Array of object/structure report objects. Descriptions of structure types newly encountered.

```json
[
  {
    "name": "Tower",
    "description": "A Tower is a ..."
  }
]
```

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Object type name |
| `description` | string | Full object description text |

### 1.15 regions

Array of region objects. Contains all regions visible to the faction this turn (present, farsight, or passed through).

Each region object is described in the next section.

### 1.16 Region Object

```json
{
  "terrain": "tundra",
  "coordinates": {
    "x": 46, "y": 84, "z": 1,
    "label": "surface",
    "depth_prefix": "deep "
  },
  "province": "Lipodou",
  "settlement": { "name": "Cityname", "size": "village" },
  "population": { "amount": 1130, "race": "gnomes" },
  "tax": 565,
  "weather": { "current": "clear", "next": "winter" },
  "description": "Description text for special regions like Nexus.",
  "wages": { "amount": 12.5, "max": 272 },
  "markets": {
    "wanted": [ /* item objects with price */ ],
    "for_sale": [ /* item objects with price */ ]
  },
  "entertainment": 26,
  "products": [ /* item objects */ ],
  "exits": [
    {
      "direction": "North",
      "region": { /* basic_region_data */ }
    }
  ],
  "gate": { "number": 510, "open": 1, "total": 100 },
  "present": true,
  "units": [ /* unit objects (units not in structures) */ ],
  "structures": [ /* structure objects */ ]
}
```

#### Coordinates

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `x` | integer | Yes | X coordinate |
| `y` | integer | Yes | Y coordinate |
| `z` | integer | Yes | Z level (1=surface, 2+=underworld) |
| `label` | string | Yes | Level name: `"surface"`, `"underworld"`, `"underdeep"`, etc. |
| `depth_prefix` | string | Optional | Depth modifier for underworld: `"deep "`, `"very deep "`, etc. |

#### Core Region Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `terrain` | string | Yes | Terrain type name (e.g., `"forest"`, `"mountain"`, `"ocean"`, `"tundra"`, `"cavern"`, `"underforest"`) |
| `province` | string | Yes | Province name |
| `settlement` | object | Optional | Town/city if present |
| `settlement.name` | string | - | Settlement name |
| `settlement.size` | string | - | `"village"`, `"town"`, or `"city"` |
| `present` | boolean | Yes | True if the faction has units physically present (not just farsight/passing) |
| `description` | string | Optional | Special description (e.g., Nexus description) |

#### Population and Economy

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `population` | object | Optional | Peasant population (absent for uninhabited regions like ocean) |
| `population.amount` | integer | - | Number of peasants |
| `population.race` | string | - | Race name (e.g., `"gnomes"`, `"wood elves"`, `"men"`) |
| `tax` | integer | Optional | Taxable silver |
| `wages` | object | Conditional | Wage information |
| `wages.amount` | float | - | Wage rate per person |
| `wages.max` | integer | Optional | Maximum total wages available |
| `entertainment` | integer | Optional | Entertainment silver available |

#### Markets

| Field | Type | Description |
|-------|------|-------------|
| `markets.wanted` | array | Items the market wants to buy (with `price`) |
| `markets.for_sale` | array | Items the market sells (with `price`) |

Market item objects include: `name`, `plural`, `tag`, `price`, and either `amount` (integer) or `unlimited` (boolean).

#### Products

Array of producible items. Each has `name`, `plural`, `tag`, and either `amount` or `unlimited`.

#### Exits

Array of exit objects. Each has:
- `direction` (string): `"North"`, `"Northeast"`, `"Southeast"`, `"South"`, `"Southwest"`, `"Northwest"`
- `region` (object): basic_region_data for the neighboring region (contains `terrain`, `coordinates`, `province`, and optionally `settlement`)

#### Gate

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `open` | integer | Yes | Whether the gate is open |
| `number` | integer | Optional | Gate number (only if open and faction can see gates) |
| `total` | integer | Optional | Total number of gates in the world (if not dispersed) |

#### Weather

| Field | Type | Description |
|-------|------|-------------|
| `current` | string | Last month's weather: `"clear"`, `"winter"`, `"monsoon season"`, `"blizzard"`, `"unnaturally clear"` |
| `next` | string | Next month's weather |

### 1.17 Unit Object

Units appear inside `region.units` (units not in structures) or `structure.units` (units inside structures).

#### Own Unit (full detail)

```json
{
  "name": "Guards",
  "number": 1264,
  "own_unit": true,
  "description": "Custom description text",
  "faction": { "name": "Distant Drummer", "number": 15 },
  "flags": {
    "guard": true,
    "avoid": false,
    "behind": false,
    "reveal": "faction",
    "holding": false,
    "taxing": true,
    "no_aid": false,
    "sharing": true,
    "consume": "faction",
    "no_cross_water": false,
    "spoils": "all"
  },
  "items": [
    { "amount": 11, "name": "gnome", "plural": "gnomes", "tag": "GNOM" },
    { "amount": 240, "name": "silver", "plural": "silver", "tag": "SILV" }
  ],
  "weight": 101,
  "capacity": { "flying": 0, "riding": 0, "walking": 77, "swimming": 0 },
  "skills": {
    "known": [
      { "name": "combat", "tag": "COMB", "level": 1, "skill_days": 30, "study_rate": 5 }
    ],
    "can_study": [
      { "name": "fire", "tag": "FIRE" }
    ]
  },
  "combat_spell": { "name": "fire", "tag": "FIRE" },
  "readied": {
    "weapons": [ { "name": "sword", "tag": "SWOR", "plural": "swords" } ],
    "armor": [ { "name": "plate armor", "tag": "PARM", "plural": "plate armors" } ],
    "item": { "name": "amulet of protection", "tag": "AMPR" }
  },
  "visited": [ "Province A", "Province B" ],
  "orders": [ /* order objects */ ]
}
```

#### Foreign Unit (limited detail)

```json
{
  "name": "City Guard",
  "number": 119,
  "attitude": "neutral",
  "faction": { "name": "The Guardsmen", "number": 1 },
  "flags": { "guard": true, "avoid": false },
  "items": [
    { "amount": 80, "name": "leader", "plural": "leaders", "tag": "LEAD" },
    { "amount": 80, "name": "sword", "plural": "swords", "tag": "SWOR" }
  ]
}
```

#### Unit Fields Reference

| Field | Type | Own Unit | Foreign Unit | Description |
|-------|------|----------|--------------|-------------|
| `name` | string | Yes | Yes | Unit name |
| `number` | integer | Yes | Yes | Unit number |
| `own_unit` | boolean | Yes (true) | Absent | Marks as own unit |
| `attitude` | string | Absent | Yes | Your attitude toward this unit: `"hostile"`, `"unfriendly"`, `"neutral"`, `"friendly"`, `"ally"` |
| `description` | string | Optional | Optional | Custom unit description |
| `faction` | object | Yes | Conditional | Faction info (foreign units: only if detected) |
| `flags` | object | Yes (full) | Partial | Unit flags (see below) |
| `items` | array | Yes (all) | Partial | Items carried (foreign: only non-zero-weight items) |
| `weight` | integer | Yes | No | Total weight carried |
| `capacity` | object | Yes | No | Carry capacities |
| `skills` | object | Yes | No | Skills known and studyable |
| `combat_spell` | object | Optional | No | Set combat spell (mages only) |
| `readied` | object | Optional | No | Readied weapons/armor/items |
| `visited` | array | Optional | No | List of province names visited |
| `orders` | array | Yes | No | Current orders |

#### Flags Detail

| Flag | Type | Own Only | Description |
|------|------|----------|-------------|
| `guard` | boolean | No | Unit is on guard |
| `avoid` | boolean | No* | Unit is avoiding combat (*visible if faction detected) |
| `behind` | boolean | Yes* | Unit fights from behind (*visible if faction detected) |
| `reveal` | string | Yes | `"unit"` or `"faction"` |
| `holding` | boolean | Yes | Unit is holding position |
| `taxing` | boolean | Yes | Unit has autotax set |
| `no_aid` | boolean | Yes | Unit refuses aid |
| `sharing` | boolean | Yes | Unit shares resources |
| `consume` | string | Yes | `"unit"` or `"faction"` |
| `no_cross_water` | boolean | Yes | Unit won't cross water |
| `spoils` | string | Yes | Battle spoils preference: `"all"`, `"weightless"`, `"flying"`, `"walking"`, `"riding"`, `"sailing"`, `"swimming"` |

#### Skills Detail

`skills.known` array entries:

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Skill name |
| `tag` | string | Skill abbreviation |
| `level` | integer | Current skill level |
| `skill_days` | integer | Skill days per man |
| `study_rate` | integer | Optional, only if REQUIRED_EXPERIENCE enabled |

`skills.can_study` array entries: `{ "name": string, "tag": string }`.

#### Capacity

| Field | Type | Description |
|-------|------|-------------|
| `flying` | integer | Flying capacity |
| `riding` | integer | Riding capacity |
| `walking` | integer | Walking capacity |
| `swimming` | integer | Swimming capacity |

#### Item Object in Unit

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Singular item name |
| `plural` | string | Yes | Plural item name |
| `tag` | string | Yes | Item abbreviation |
| `amount` | integer | Conditional | Quantity (for normal items) |
| `unfinished` | boolean | Conditional | True for incomplete ship items |
| `needs` | integer | Conditional | Points still needed (for unfinished ships) |
| `illusion` | boolean | Optional | True if item is illusory (visible with truesight) |

Items are reported in a specific order: men, monsters, weapons/combat items, mounts, wagons, other items, silver.

### 1.18 Structure Object

Structures appear in `region.structures` array.

```json
{
  "name": "Infidel",
  "number": 100,
  "type": "Fleet",
  "description": "Custom description",
  "fleet": true,
  "ships": [
    { "name": "Longship", "number": 1, "plural": "Longships" }
  ],
  "load": 120,
  "capacity": 150,
  "sailors": 8,
  "fleet_size": 4,
  "max_speed": 4,
  "damage_percent": 0,
  "sail_directions": { "N": true, "SE": true },
  "units": [ /* unit objects */ ]
}
```

#### Building Structure

```json
{
  "name": "Tower of Power",
  "number": 42,
  "type": "Tower",
  "incomplete": 10,
  "inner_location": true,
  "warding_runes": true,
  "closed": true,
  "decaying": true,
  "needs_maintenance": true,
  "sacrifice": {
    "name": "silver", "tag": "SILV", "plural": "silver", "amount": 100
  },
  "grantskill": { "name": "combat", "tag": "COMB", "level": 1 },
  "units": [ /* unit objects */ ]
}
```

#### Structure Fields Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Structure name |
| `number` | integer | Yes | Structure number |
| `type` | string | Yes | Structure type name (e.g., `"Tower"`, `"Fleet"`, `"Castle"`) |
| `description` | string | Optional | Custom description |
| `units` | array | Yes | Units inside this structure |

**Fleet-specific fields** (when `fleet` is true):

| Field | Type | Description |
|-------|------|-------------|
| `fleet` | boolean | Always true for fleets |
| `ships` | array | Ship items: `{ name, number, plural }` |
| `load` | integer | Current cargo weight |
| `capacity` | integer | Maximum cargo capacity |
| `sailors` | integer | Current sailing skill total |
| `fleet_size` | integer | Required sailing skill |
| `max_speed` | integer | Maximum fleet speed |
| `damage_percent` | integer | Damage percentage (0-100) |
| `sail_directions` | object | Available sail directions (keys are direction abbreviations) |

**Building-specific fields:**

| Field | Type | Description |
|-------|------|-------------|
| `incomplete` | integer | Construction points remaining |
| `inner_location` | boolean | Contains an inner location |
| `warding_runes` | boolean | Has Runes of Warding engraved |
| `closed` | boolean | Cannot be entered by players |
| `decaying` | boolean | About to decay |
| `needs_maintenance` | boolean | Needs maintenance |
| `sacrifice` | object | Required sacrifice item (with `name`, `tag`, `plural`, `amount`) |
| `grantskill` | object | Skill granted to owner: `{ name, tag, level }` |

### 1.19 Item Object

Items appear in various contexts (unit items, market items, products, statistics). The fields vary by context:

| Field | Context | Type | Description |
|-------|---------|------|-------------|
| `name` | All | string | Singular name |
| `plural` | All | string | Plural name |
| `tag` | All | string | 4-letter abbreviation |
| `amount` | Most | integer | Quantity |
| `unlimited` | Markets/Products | boolean | Unlimited supply |
| `price` | Markets | integer | Buy/sell price |
| `unfinished` | Unit items | boolean | Unfinished ship |
| `needs` | Unit items | integer | Points needed to complete |
| `illusion` | Unit items | boolean | Illusory item |

### 1.20 Orders Structure

Orders appear in `unit.orders` as an array. Each order is an object with an `order` field (string) and optionally a `nested` field (array of orders) for TURN/FORM blocks.

```json
[
  { "order": "@SHARE 0" },
  { "order": "@AVOID 1" },
  { "order": "MOVE NW" },
  {
    "order": "@TURN",
    "nested": [
      { "order": "MOVE SE" },
      { "order": "TAKE FROM 2638 ALL HORS" }
    ]
  },
  { "order": "ENDTURN" }
]
```

The `@` prefix indicates a repeating order. TURN/ENDTURN blocks contain deferred orders. FORM/ENDFORM blocks can also be nested.

## 2. Text Report Structure

The text report is generated by `TextReportGenerator::output()` from the JSON report data. Lines are wrapped at 70 characters with 2-space indentation for continuation lines.

### 2.1 Section Order

The sections appear in this exact order:

1. Treasury (statistics) -- commented with `;` prefix
2. Report header (faction name, number, type, date)
3. Engine version
4. Administrative messages (Times, password, inactivity, quit)
5. Faction Status
6. Errors during turn
7. Battles during turn
8. Events during turn
9. Skill reports
10. Item reports
11. Object reports
12. Declared Attitudes
13. Unclaimed silver
14. Regions (with units and structures)

### 2.2 Treasury (Statistics)

Appears as comments at the very top of the report, before all other content.

```
;Treasury:
;
;Item                                      Rank  Max        Total
;=====================================================================
;6 leaders [LEAD]                          27    21         385
;10 wood elves [WELF]                      27    399        2974
;1272 silver [SILV]                        50    54164      577289
```

Each line is prefixed with `;`. Columns are: item string (42 chars), rank (6 chars), max (11 chars), total.

### 2.3 Report Header

```
Atlantis Report For:
Distant Drummer (15) (Martial 1, Magic 4)
April, Year 2
```

Format: `Name (Number) (Type1 Points1, Type2 Points2)`. The type section shows capitalized type names with their point values. Month and year on the next line.

### 2.4 Engine Version

```
Atlantis Engine Version: 5.2.5 (beta)
NewOrigins, Version: 3.0.0 (beta)
```

### 2.5 Administrative Messages

These appear conditionally:

- `Note: The Times is not being sent to you.` -- if times_sent is false
- `REMINDER: You have not set a password for your faction!` -- if password_unset
- `WARNING: You have N turns until your faction is automatically removed due to inactivity!` -- if near deletion
- Various quit messages (restart, game over, won, eliminated)

### 2.6 Faction Status

```
Faction Status:
Regions: 3 (10)
Quartermasters: 0 (2)
Mages: 4 (5)
Apprentices: 0 (10)
```

Format: `Category: current (allowed)`. Only categories relevant to the game configuration appear.

### 2.7 Errors

```
Errors during turn:
Guards (2171): GIVE: Nonexistant target (2996).
Builders (2896): STUDY: Not enough funds.
```

Format: `UnitName (UnitNumber): ErrorMessage` or just `ErrorMessage` if no unit.

### 2.8 Battles

```
Battles during turn:
The Diplomat (1445) attacks Informator (3655) in mountain (43,81) in Derngill!

Attackers:
Unit A (123), Faction (1), behind, 10 humans [HUMN], combat 2.
...

Defenders:
Unit B (456), Faction (2), 5 orcs [ORC].
...

Round 1:
...
```

Battle reports are output as pre-formatted text lines from the `battle.report` JSON array. Each line is printed verbatim.

### 2.9 Events

```
Events during turn:
Times reward of 200 silver.
Guards (1264): Collects $550 in taxes in tundra (46,84) in Lipodou.
Masked (1263) steals 0 silver [SILV] from Shar Ilbechin (1625).
```

Format: `UnitName (UnitNumber): Message` or just `Message` if no unit. Long lines wrap with 2-space indentation.

### 2.10 Skill Reports

```
Skill reports:

sailing [SAIL] 2: No skill report.

summon skeletons [SUSK] 2: A mage with this skill may summon ...
```

Format: `skillname [TAG] level: description`. Each skill report is preceded by a blank line.

### 2.11 Item Reports

```
Item reports:

A sword [SWOR] is a basic weapon...
```

Each item description is preceded by a blank line. The description is a self-contained paragraph.

### 2.12 Object Reports

```
Object reports:

Tower: A Tower is a hardened structure...
```

Format: `ObjectTypeName: description`. Each preceded by a blank line.

### 2.13 Attitudes

```
Declared Attitudes (default Unfriendly):
Hostile : none.
Unfriendly : Creatures (2).
Neutral : none.
Friendly : none.
Ally : none.
```

The default attitude is capitalized in the header. Each attitude level lists factions or `none`. Attitudes always appear in order: Hostile, Unfriendly, Neutral, Friendly, Ally.

### 2.14 Unclaimed Silver

```
Unclaimed silver: 403.
```

### 2.15 Region Format

```
forest (40,84) in Tifufirth, 1768 peasants (wood elves), $1131.
------------------------------------------------------------
  Wages: $13.2 (Max: $484).
  Wanted: none.
  For Sale: 70 wood elves [WELF] at $42, 14 leaders [LEAD] at $739.
  Entertainment available: $58.
  Products: 32 grain [GRAI], 25 wood [WOOD], 13 furs [FUR].

Exits:
  North : ocean (40,82) in Atlantis Ocean.
  Northeast : ocean (41,83) in Atlantis Ocean.

There is a Gate here (Gate 510 of 100).

* Unit1 (1258), Faction (15), ...
- Unit2 (119), on guard, Other Faction (1), ...

+ Infidel [100] : Longship; 0% damaged; Load: 120/150; ...
  * Unit3 (1851), ...
```

**Region header line format:**
`terrain (x,y) in Province, N peasants (race), $tax.`

For underworld regions:
`cavern (x,y,underworld) in Province, contains Settlement [size], N peasants (race), $tax.`

With `EASIER_UNDERWORLD` disabled, the z coordinate and depth prefix are shown:
`cavern (x,y,z <depth_prefix label>) in Province`

**Weather line:**
`The weather was clear last month; it will be winter next month.`
or `It was winter last month; it will be clear next month.`
or `There was an unnatural blizzard last month; ...`

**Separator:** `------------------------------------------------------------` (60 dashes)

**Indented section** (2 spaces): Wages, Wanted, For Sale, Entertainment, Products.

**Gate formats:**
- `There is a Gate here (Gate 510 of 100).` -- open gate with total
- `There is a Gate here (Gate 510).` -- open gate, dispersed numbers
- `There is a closed Gate here.` -- closed gate

### 2.16 Unit Format

```
* Glum (1258), Distant Drummer (15), avoiding, behind, leader [LEAD],
  mithril sword [MSWO]. Weight: 11. Capacity: 0/0/15/0. Skills: force
  [FORC] 1 (35), pattern [PATT] 2 (110). Combat spell: fire [FIRE].
  Can Study: fire [FIRE], earthquake [EQUA]. Has visited Province A
  and Province B; Custom description.
```

**Unit line components** (in order):

1. **Prefix symbol** (see [Section 2.18](#218-unit-attitude-symbols))
2. **Name and number:** `UnitName (Number)`
3. **Guard flag** (before faction): `, on guard`
4. **Faction** (if visible): `, FactionName (FactionNumber)`
5. **Flags** (in order): `, avoiding`, `, behind`, `, revealing unit/faction`, `, holding`, `, taxing`, `, receiving no aid`, `, sharing`, `, consuming unit's/faction's food`, `, won't cross water`, `, TYPE battle spoils` (if not "all")
6. **Items:** comma-separated list: `N itemname [TAG]` (singular for 1, plural for >1)
7. **Weight:** `. Weight: N`
8. **Capacity:** `. Capacity: flying/riding/walking/swimming`
9. **Skills:** `. Skills: skillname [TAG] level (days)` or `. Skills: none`
   - With experience: `skillname [TAG] level (days+rate)`
10. **Combat spell:** `. Combat spell: name [TAG]`
11. **Readied weapons:** `. Ready weapon: name [TAG]` (or `Ready weapons:` for multiple)
12. **Readied armor:** `. Ready armor: name [TAG]`
13. **Readied item:** `. Ready item: name [TAG]`
14. **Can Study:** `. Can Study: skill1 [TAG], skill2 [TAG]`
15. **Visited:** `. Has visited Province1, Province2 and Province3`
16. **Description:** `; Custom description text`
17. **Period:** `.`

### 2.17 Structure/Object Format

**Building:**
```
+ Tower of Power [42] : Tower, needs 10.
  * Unit1 (100), ...
```

Format: `+ StructureName [Number] : TypeName`

Optional suffixes (in output order):
- `, needs N` -- incomplete construction
- `, about to decay` -- decaying
- `, needs maintenance` -- needs maintenance
- `, contains an inner location`
- `, engraved with Runes of Warding`
- `; description text` -- custom description (note: semicolon, not comma)
- `, closed to player units`
- `, requires sacrifice of N itemname [TAG]`
- `, grants owner skillname [TAG] level`

**Fleet:**
```
+ Infidel [100] : Longship; 0% damaged; Load: 120/150; Sailors: 8/4;
  MaxSpeed: 4; Sail directions: N, SE.
  * Sea Scouts (1851), ...
```

For fleets with a single ship type, the type name is the ship name directly. For multiple ship types: `+ FleetName [Number] : Fleet, N ShipType1, M ShipType2`.

Fleet suffixes (semicolon-separated):
- `; N% damaged`
- `; Load: current/capacity`
- `; Sailors: current/required`
- `; MaxSpeed: speed`
- `; Sail directions: dir1, dir2`
- `; description text`

Units inside structures are indented by 2 additional spaces under the structure header.

### 2.18 Unit Attitude Symbols

The prefix character before each unit line indicates the relationship:

| Symbol | Meaning | Description |
|--------|---------|-------------|
| `*` | Own unit | Unit belongs to your faction |
| `=` | Ally | Unit's faction declared as ally |
| `:` | Friendly | Unit's faction declared as friendly |
| `-` | Neutral | Unit's faction is neutral (also default for unknown) |
| `%` | Unfriendly | Unit's faction declared as unfriendly |
| `!` | Hostile | Unit's faction declared as hostile |

Note: Attitude symbols for foreign units only appear when `show_unit_attitudes` is enabled in administrative settings. Otherwise all foreign units show `-`.

## 3. Orders Template Structure

The orders template is a separate output file (not part of the report) that provides a pre-formatted template for submitting next turn's orders.

### 3.1 Template Types

Three formats exist: Short, Long, and Map.

All templates begin with:
```
Orders Template (Long Format):

#atlantis FACTION_NUMBER "PASSWORD"
```

And end with:
```
#end
```

### 3.2 Short/Long Format

**Short format:** Only `unit NUMBER` headers with existing orders, no unit details.

**Long format:** Includes commented-out unit descriptions:
```
;*** tundra (46,84) in Lipodou ***

unit 1258
;Glum (1258), avoiding, behind, leader [LEAD], mithril sword [MSWO].
;  Weight: 11. Capacity: 0/0/15/0. Skills: force [FORC] 1 (35), ...
```

Region headers use `*** terrain (x,y) in Province ***` format.
Unit descriptions are commented with `;` prefix and do NOT show faction info (only unit's own perspective).

### 3.3 Map Format

The Map format includes an ASCII hex map alongside region data:

```
;-----------------------------------------------------------
;forest (40,84) in Tifufirth
;         ____          Next winter
;  nw     /    \     ne  Tax   1131
;   ____/  ^ ^  \____    Ente    58
;  /    \  ^ ^  /    \   Wage   13.2 (max 484)
; /      \____/      \
; \      /    \      /  Want  192 GRAI @ 27
;  \____/ ~ ~  \____/        124 LIVE @ 27
;  /    \ ~ ~  /    \
; /      \____/      \  Sell   70 WELF @  42
; \      /    \      /        14 LEAD @ 739
;  \____/      \____/
;       \      /         Prod   32 GRAI
;  sw    \____/     se        25 WOOD
```

The hex map shows terrain fills for the center hex and all 6 neighboring hexes. Settlement names replace terrain fill when present.

Terrain fill patterns:
- ocean: `~ ~` / `~ ~`
- plain: (blank)
- forest: `^ ^` / `^ ^`
- mountain: `/\/\` / `/  \ \`
- swamp: `v v` / `v v`
- jungle: `@ @` / `@ @`
- desert: `. .` / `. .`
- tundra: `' '` / `' '`

Right side shows: weather, tax, entertainment, wages, wanted items, for sale items, products, gate info.
